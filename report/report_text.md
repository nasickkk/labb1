# Текст для отчета по лабораторной работе № 1, вариант 1

## Введение

Целью лабораторной работы является изучение возможностей языка Python по обработке данных и создание веб-приложения с использованием фреймворка Flask. В рамках выполнения работы также рассматриваются инструменты системы контроля версий Git и настройка непрерывной интеграции с использованием AppVeyor.

Практическая часть направлена на разработку веб-приложения, которое выполняет обработку загруженного изображения. Согласно варианту 1, приложение должно устранять шум изображения по заданному пользователем параметру сглаживания, а также строить график распределения цветов и график распределения шума.

## 2 Текст задания

Во всех вариантах лабораторной работы № 1 необходимо разработать веб-приложение с использованием фреймворка Flask, системы контроля версий Git и системы непрерывной интеграции AppVeyor. Каждое приложение должно обеспечивать проверку на робота с помощью капчи или любой другой технологии. Объекты отображения и ввода должны быть размещены удобно для пользователя.

Вариант 1: веб-приложение должно обеспечивать преобразование подаваемой на вход картинки путем устранения шума в соответствии с переданным параметром сглаживания или любого другого параметра. Метод фильтрации можно выбрать самостоятельно. Должна быть возможность вывода графика распределения цветов на картинке и графика распределения шума.

## 3 Решение задачи

### 3.1 Создание репозитория на GitHub

Для хранения исходного кода был создан репозиторий на GitHub. В репозиторий добавлены основные файлы проекта: `app.py`, `requirements.txt`, `appveyor.yml`, HTML-шаблоны в папке `templates` и статические файлы в папке `static`.

Структура проекта:

```text
lr1_variant1_complete/
├── app.py
├── appveyor.yml
├── requirements.txt
├── README.md
├── templates/
│   ├── index.html
│   └── result.html
└── static/
    ├── style.css
    ├── uploads/
    └── results/
```

### 3.2 Настройка непрерывной интеграции AppVeyor

Для автоматизации проверки проекта был добавлен файл `appveyor.yml`. В нем указано, что AppVeyor должен создать среду Windows, установить Python, загрузить зависимости из `requirements.txt`, выполнить проверку синтаксиса файла `app.py` и проверить импорт Flask-приложения.

### 3.3 Развертывание приложения на PythonAnywhere

Для развертывания приложения можно использовать сервис PythonAnywhere. В консоли сервиса необходимо клонировать репозиторий, установить зависимости командой `pip install -r requirements.txt`, затем в разделе Web указать путь к проекту и настроить WSGI-файл:

```python
import sys
path = '/home/USERNAME/lr1_variant1_complete'
if path not in sys.path:
    sys.path.append(path)

from app import app as application
```

После сохранения настроек и перезапуска веб-приложение становится доступно по адресу, который выдает PythonAnywhere.

### 3.4 Результат выполнения работы

В результате было разработано Flask-приложение, которое позволяет пользователю загрузить изображение, пройти проверку с помощью капчи, выбрать метод фильтрации и задать параметр сглаживания. После обработки приложение выводит исходное изображение, обработанное изображение, карту шума, график распределения цветов и график распределения шума.

Дополнительно подготовлен файл конфигурации AppVeyor для автоматической проверки проекта.
